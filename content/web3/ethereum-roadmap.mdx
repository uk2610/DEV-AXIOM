---

## Stage 1: EVM Core

- EVM execution model and opcodes (high level)
- Storage slots, calldata, memory vs storage
- ABI encoding and event indexing

### Must-Do
- Write and deploy a contract with event emission
- Explain transaction receipt fields in interviews

## Stage 2: Solidity Engineering

- Contract inheritance and composition
- Modifiers, custom errors, and access control
- Upgradeability strategies and proxy patterns

### Must-Do
- Build a role-based contract with tests
- Add custom errors and benchmark gas savings

## Stage 3: App + Infra

- Frontend contract integration patterns
- Backend indexing with logs and pagination
- Monitoring failures and reorg-safe workflows

### Must-Do
- Build a dashboard that tracks user actions from events
- Implement retry-safe job processing for chain reads

## Stage 4: Advanced Readiness

- L2 ecosystem decisions
- MEV awareness and transaction strategy
- Security checklist and audit prep

### Interview Prompts
- "How would you design a staking system with withdrawal delays?"
- "What breaks if an external call re-enters?"
- "How do you make event-driven systems reliable?"

## Final Output

Ship one mini protocol with:
- token or accounting logic
- tested edge cases
- simple interface
- design rationale doc
