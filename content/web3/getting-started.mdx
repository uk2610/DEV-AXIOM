## Why This Roadmap Works

Most Web3 roadmaps fail because they are topic lists, not execution systems. This one is phase-based and output-driven.

## Phase 1: Foundation (Weeks 1-4)

### Concepts
- Blockchain anatomy: blocks, state, mempool, finality
- Accounts, wallets, keys, signatures
- Gas, fees, transactions, nonce management
- RPC providers and node roles

### Output Goals
- Explain full transaction lifecycle in your own words
- Build a wallet flow in a sample app
- Compare account models: EVM vs Solana

## Phase 2: Builder Mode (Weeks 5-8)

### Concepts
- Smart contract design and storage patterns
- Token standards and events
- Off-chain indexing and backend integration
- Testing strategy and local simulation

### Output Goals
- Deploy one minimal contract + integration UI
- Write tests for state transitions and edge cases
- Build a simple dashboard for contract events

## Phase 3: Interview Readiness (Weeks 9-12)

### Concepts
- Security vulnerabilities and threat modeling
- Scalability patterns and data consistency
- Protocol tradeoffs and architecture storytelling

### Output Goals
- Conduct one mock design interview per week
- Build one mini protocol case-study writeup
- Present system design decisions with pros/cons

## Weekly Cadence

- `Mon-Tue`: Learn + annotate docs
- `Wed-Thu`: Build and test
- `Fri`: Timed interview simulation
- `Sat`: Review failures and re-implement weak areas

## Success Metrics

- You can explain architecture under time pressure
- You can ship a working mini dApp end-to-end
- You can reason about security and scale tradeoffs
